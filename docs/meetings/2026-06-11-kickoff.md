## 1. 완성도 기준 — "차트의 기민함"

> 가장 중요한 품질 기준으로 합의.

- 지표 스킨이 **차트 view 변화(스크롤·줌·기간 변경)에 딱딱 정확히 따라붙어야** 함.
- 좌표 추적이 어긋나거나 끊기면 완성도가 무너짐 → **1순위 품질 지표**로 관리.

> 📌 POC 연계: 현재 POC는 가격→픽셀 좌표 변환 + view 변경 시 재계산을 구현해 둠. 이 "기민함"을 벤치마크 기준선으로 사용.

---

## 2. 마켓플레이스 테마 개발 방향

핵심은 **"쓰고 싶은, 맛있는" 콘텐츠** — 배경 스킨 + 지표 스킨 조합.

### 지표 스킨 아이디어

- **일목균형표 → 진짜 "구름"으로 시각화**
  - 하늘 배경에 일목균형표를 구름처럼 렌더링
  - 가격이 구름을 돌파할 때 **비행기가 구름을 뚫고 나가는 듯한** 연출
- 사람들이 **많이 쓰는 지표** 위주로 우선 제공
- **골든크로스 / 데드크로스 → 교차 지점 반짝이는** 이펙트

---

## 3. 제공 기능 정의

### 3-1. 지표 스킨
제공할 지표 목록 (→ **우선순위 확정 필요**):
- 일목균형표
- 골든크로스 / 데드크로스
- 최고점 / 최저점
- 지지선 / 저항선
- 매물대

### 3-2. 배경 스킨
- **DIY 스킨 만들기** (사용자 직접 제작)

### 3-3. 위젯 (신규 논의 영역)
- 차트 테마 외 **위젯류**에 대한 고민 필요
- 예: **고양이 위젯** — 화면을 그냥 뛰어다니는 고양이

### 3-4. 커스텀 스킨 (AI)
- 사용자가 **자연어로 입력** → 배경·지표 아이콘 자동 생성
- **AI API 연결** 기능
