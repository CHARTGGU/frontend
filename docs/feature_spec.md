# ChartSkin — 프로젝트 피처 스펙

> 차트 위에 스킨(테마)을 입혀서 나만의 트레이딩 뷰를 만드는 웹앱

---

## 제품 한 줄 정의

> 주식·코인 차트를 불러오고, 마켓플레이스에서 테마를 골라 입히는 웹앱.
> 배경·캐릭터·말풍선 등 감성적인 스킨으로 차트를 꾸미고, AI로 나만의 스킨을 만든다.

---

## 전체 화면 구조

```
┌────────────────────────────────────────────┬──────────────────┐
│                                            │                  │
│   헤더: 종목 검색 / 마켓 선택               │   사이드바        │
│                                            │   마켓플레이스    │
├────────────────────────────────────────────│                  │
│                                            │  [테마 카드들]   │
│              차트 영역                      │  - 배경 스킨     │
│                                            │  - 지표 스킨     │
│   ┌─────────────────────────────────────┐  │  - 세트 테마     │
│   │  배경 스킨 레이어 (고정)              │  │                  │
│   │  + 차트 캔들/라인                    │  │  [적용 중인 스킨] │
│   │  + 지표 스킨 레이어 (좌표 추적)      │  │  - 수정 / 해제   │

│                                            │                  │
│   거래량 차트                               │                  │
└────────────────────────────────────────────┴──────────────────┘
```

## 스킨 타입 정의

### 1. 배경 스킨 (Background Skin)
- 차트 뒤에 깔리는 이미지/일러스트
- 차트 스크롤·줌에 **고정** (움직이지 않음)
- 투명도 조절 가능 (차트 가독성 확보)
- 예시: 밤하늘, 장원영 사진, 귀여운 패턴

### 2. 지표 스킨 (Indicator Skin)
- 특정 **가격 포인트**에 캐릭터/아이콘을 붙이는 스킨
- 차트 스크롤·줌 시 **좌표 추적** (가격축 연동)
- 바인딩 포인트 종류:
  - 기간 내 최고점 / 최저점
  - 매수 / 매도 지점 (사용자 직접 지정)
  - 이동평균선 골든크로스 / 데드크로스
  - **일목균형표 (구름)** — 선행스팬 사이 영역을 구름으로 표현
  - **지지선 / 저항선**
  - **매물대 (Volume Profile)**
  - 사용자 정의 가격선
- 캐릭터 상태: 상승 시 happy, 하락 시 sad, 횡보 시 neutral (표정 variant)
- 말풍선 텍스트 커스터마이징 가능
- **연출 아이디어**:
  - 일목균형표를 하늘 배경 위 진짜 "구름"으로 그리고, 가격이 구름을 돌파할 때 **비행기가 뚫는** 연출
  - 골든크로스 / 데드크로스 지점 **반짝임** 이펙트

### 3. 위젯 (Widget)
- 차트 좌표와 무관하게 화면에 얹는 부가 요소 (지표 바인딩 X)
- 예시:
  - **고양이** — 차트 위를 그냥 뛰어다니는 캐릭터 (감성 요소)
  - **뉴스 마커** — 종목 뉴스를 차트 시간축에 표시
- 배경/지표 스킨과 독립적으로 켜고 끔

### 4. 세트 테마 (Theme Set)
- 배경 스킨 + 지표 스킨 + 위젯 + 차트 색상 + 말풍선 스타일을 묶은 패키지
- 예시: "밤하늘 테마", "귀엽고 달달한 테마", "사이버펑크 테마"

## 피처 목록

### Phase 1 — MVP (실제 1차 구현 범위)

> 실 구현 범위·작업 단위는 [`mvp.md`](./mvp.md) 작업 분해(M0~M7)와 1:1 대응.
> **MVP는 Binance(암호화폐) 단독 + 박아둔 스킨 토글** — 주식·공유 마켓플레이스는 Backlog로 미룸.

#### 1-0. 데이터 파이프라인 (선행 필수 · M1)
> ⚠️ lightweight-charts는 시세 0 제공 → 모든 차트 피처의 전제.
- [ ] Binance REST 클라이언트 (`/api/v3/klines`) — 키 불필요·CORS 허용, 브라우저 직접 호출
- [ ] 응답 → 통일 `Candle{time,open,high,low,close,volume}` 어댑터
- [ ] interval 파라미터 (1m/5m/15m/1h/1d…)
- [ ] 심볼 검색/선택 (고정 목록 또는 `/exchangeInfo`)
- [ ] 로딩·에러 상태

#### 1-1. 차트 뷰어 (M2)
- [ ] lightweight-charts **v5** 마운트 (`'use client'`, `ssr:false`) + 리사이즈
- [ ] 캔들차트 + 거래량 히스토그램(별도 pane)
- [ ] 이동평균선 (5/20/60/120) — **SMA 직접 계산** 후 LineSeries 주입
- [ ] 기간 선택 (1m/5m/15m/1h/1d…) ↔ refetch ↔ `setData`
- [ ] 줌, 스크롤, 십자선 커서 (라이브러리 내장)
- [ ] 차트 배경 투명화 (배경 스킨 레이어 준비)
- [ ] ~~마켓 선택: 국내/미국 주식~~ → **Backlog** (API 키 → 백엔드 필요)

#### 1-2. 스킨 전환 UI (사이드바 · M6)
> MVP는 **앱에 박아둔 프리셋 스킨** 토글. 공유 마켓플레이스(DB)는 Backlog.
- [ ] 프리셋 스킨 카드 목록 (썸네일 + 이름)
- [ ] 적용 / 해제 토글 → `skinStore`
- [ ] 현재 적용 중인 스킨 표시
- [ ] (옵션) localStorage로 마지막 선택 유지
- [ ] ~~카테고리/태그 필터, 공유 테마 목록~~ → **Backlog** (공유 데이터 = DB+API)

#### 1-3. 배경 스킨 렌더링 (M4)
- [ ] 번들 정적 이미지 1~2개를 차트 뒤 레이어에 렌더 (스크롤·줌 고정)
- [ ] 투명도 슬라이더
- [ ] 이미지 fit 모드: cover / contain / tile

#### 1-4. 지표 스킨 렌더링 (핵심 · M3+M5)
- [ ] 좌표 오버레이 엔진: `priceToCoordinate`/`timeToCoordinate` 래핑 (`useChartOverlay`)
- [ ] 스크롤·줌·리사이즈 구독 → rAF throttle 재배치 (**기민함 = 완성도 핵심**)
- [ ] 가시영역 밖(`null` 좌표) 숨김 처리
- [ ] 최고점 / 최저점 자동 감지 → 캐릭터 표시
- [ ] 말풍선 텍스트 기본 제공

#### 1-5. 차트 내보내기 (M7 · ⚠️ 조기 PoC)

## 확정된 결정 사항

| 항목 | 결정 |
|------|------|
| 스킨 저장 위치 | 로컬 스토리지 (서버 불필요) |
| KRX 데이터 | 전일 종가 데이터로 충분 (실시간 불필요) |
| 마켓플레이스 초기 콘텐츠 | 직접 제작 프리셋 스킨 10개로 시작 |
| 사용자 계정 | 로그인 없음 |
| 모바일 대응 | 데스크탑 전용 (반응형 불필요) |

→ **MVP(암호화폐 단독)는 백엔드 거의 안 탐** — Binance 직접 호출로 충분.
> 단 프레임워크는 처음부터 **Next.js 풀스택**으로 출발 (Route Handlers = 백엔드 자리 확보). backlog의 주식·AI·스킨공유가 들어오면 그대로 백엔드로 확장 → 마이그레이션 0. 근거: `tech_stack.md` §3.0.

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프레임워크 | **Next.js (App Router) + TypeScript** — 프론트+백엔드 단일 코드베이스 |
| 차트 | TradingView Lightweight Charts v5 |
| 스킨 렌더링 | DOM 오버레이 (Phase 2 Lottie/GIF 대비) — `tech_stack.md` §2-B |
| 스타일링 | Tailwind CSS |
| 상태관리 | Zustand |
| 백엔드 | **Next.js Route Handlers (`app/api/*`)** — 시세 프록시·키 보호·DB·AI 프록시 |
| 스킨 저장 | MVP: localStorage(메타). 업로드/AI 이미지: S3·R2 + DB |
| AI (Phase 3) | Replicate / Stability / DALL·E (백엔드 프록시) + rembg(Python) |

> MVP(암호화폐 단독)는 브라우저 직접 호출이라 백엔드 거의 안 탐. 주식·AI·스킨공유가 들어오면 Route Handlers가 그대로 백엔드. 상세 근거: `tech_stack.md` §3.0.
